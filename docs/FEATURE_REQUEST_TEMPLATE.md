## Summary

_One sentence describing what you want._

> Example: "I want to set a custom invoice number prefix so my invoices follow my existing numbering system (e.g. OFG-2026-001)."

---

## The Problem You're Solving

_Describe the real-world situation that prompted this request. What are you trying to accomplish? What goes wrong with the current product?_

> Right now, PortalFlow auto-generates invoice numbers that don't match my existing numbering scheme. When I send these to clients, they don't match my QuickBooks records and I have to manually cross-reference. This adds 10–15 minutes per invoice.

---

## Your Current Workaround

_How are you handling this today? Is there a workaround, and how painful is it?_

> I manually note the real invoice number in the Notes field, but this is easy to forget and looks unprofessional in the client portal.

---

## Proposed Solution

_If you have a specific implementation in mind, describe it. If not, skip this section — describing the problem clearly is enough._

> Add a "Invoice number" text field to the new invoice form, prefilled with the next number in sequence but editable. Allow each org to set a default prefix in Settings.
